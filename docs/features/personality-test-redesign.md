## 1. 설계 방향

### 1.1 기존 방식의 한계

현재 구현(`PersonalityTest.tsx`)은 세 성향 시스템을 각각 독립된 스텝으로 직접 선택하게 한다.

| 문제 | 설명 |
|------|------|
| 의도 노출 | "Step 1 · MBTI" 레이블이 바로 보이므로 유저가 원하는 유형을 역산해 선택함 |
| 에니어그램 무효화 | 9개 카드 중 직접 클릭은 자기 인식 편향이 강하게 작용함 |
| D&D 성향 무효화 | 3×3 그리드 직접 선택은 "선" 쪽이 더 매력적으로 보이는 후광효과 발생 |
| 측정 정밀도 없음 | 각 축이 질문 1개로 결정되어 노이즈 보정 불가 |
| 에니어그램 4번 취약 | 카드 설명이 긍정적으로 보여 의도적 선택 과다 발생 |

### 1.2 서사 시나리오 방식 선택 이유

**핵심 원칙**: 유저는 판타지 세계 속 인물로서 행동을 선택하고, 그 선택의 심리적 배경이 성향 지표로 변환된다.

- **의도 은닉**: 선택지 레이블이 "E형 행동"이 아닌 서사적 행동 묘사이므로 역산이 어려움
- **동기 측정**: 에니어그램은 행동이 아닌 동기로 구분해야 하는데, 서사 씬은 "왜 그렇게 하는가"를 자연스럽게 유도함
- **중립적 선택지**: 4가지 선택지가 모두 해당 상황에서 합리적으로 보이도록 설계
- **복합 측정**: 하나의 씬이 MBTI 2개 축 + 에니어그램 3개 유형 + D&D 1개 축을 동시에 측정

### 1.3 씬 구성 세계관

> 당신은 "아스트라 대륙" 어딘가에 있는 여행자입니다. 모험 길 위에서 마주치는 12가지 상황 속 선택이 당신이 누구인지를 결정합니다.

## 2. 3대 성향 시스템 측정 설계

### 2.1 MBTI — 4개 이분법 축

| 축 | 양극 | 측정 전략 |
|----|------|-----------|
| E/I | 외향(E) vs 내향(I) | 사회적 상황 vs 단독 행동 선호, 에너지 충전 방식 |
| S/N | 감각(S) vs 직관(N) | 구체적 정보 처리 vs 패턴/가능성 우선 |
| T/F | 사고(T) vs 감정(F) | 논리/효율 vs 관계/공감 중심 판단 |
| J/P | 판단(J) vs 인식(P) | 계획/결단 vs 유연성/탐색 선호 |

**점수 체계**: 각 축은 씬당 -2~+2 범위, 총합의 부호로 유형 결정

### 2.2 에니어그램 — 9유형 카운터

**핵심 원칙**: 행동의 표면이 아닌 내면 동기를 묻는다.

| 유형 | 핵심 동기 | 씬 설계 시 포착 방식 |
|------|-----------|----------------------|
| 1 | 옳고 그름, 내적 기준 준수 | "잘못된 것을 바로잡고 싶어서" 행동하는 선택지 |
| 2 | 사랑받음, 타인을 위한 희생 | "도움을 줘야 인정받을 수 있어서" 행동하는 선택지 |
| 3 | 성공, 가치있어 보이는 이미지 | "성과를 보여줘야 하기 때문에" 행동하는 선택지 |
| 4 | 독특함, 자아 정체성 | "나만의 방식이어야 의미가 있어서" 행동하는 선택지 |
| 5 | 지식, 역량, 독립적 이해 | "충분히 이해한 다음에 결정해야 해서" 행동하는 선택지 |
| 6 | 안전, 권위/규칙 신뢰 | "이게 옳은 절차이기 때문에" 행동하는 선택지 |
| 7 | 즐거움, 고통/제한 회피 | "더 재미있는 가능성을 놓치기 싫어서" 행동하는 선택지 |
| 8 | 통제권, 자율성 | "내가 직접 결정하고 주도해야 하기 때문에" 행동하는 선택지 |
| 9 | 갈등 회피, 조화 | "아무도 상처받지 않았으면 해서" 행동하는 선택지 |

**4번 유형 강화 전략**: 씬 3, 7, 9, 10, 12에서 "집단과 다른 나만의 독특한 선택"을 고점수로 배치.

### 2.3 D&D 성향 — 2축 독립 측정

| 축 | 양극 | 측정 전략 |
|----|------|-----------|
| L/C축 | 질서(L) vs 혼돈(C) | 규칙/절차 존중 vs 상황 판단 우선 |
| G/E축 | 선(G) vs 악(E) | 타인/공동체 배려 vs 자기 이익 우선 |

**임계값**: 각 축 합산 ±4 이상 → Lawful/Chaotic/Good/Evil, 그 사이 → Neutral
**중요**: 두 축은 완전히 독립적으로 측정된다.

## 3. 12개 씬 전체 설계

> **스코어 표기법**
> - MBTI: `E+2`, `I+1`, `S+2`, `N+1`, `T+2`, `F+1`, `J+2`, `P+1`
> - 에니어그램: `Enn1+3`, `Enn4+2` 등 (9개 유형 각각의 누적 카운터)
> - D&D L/C축: `L+2` (Lawful), `C+2` (Chaotic)
> - D&D G/E축: `G+2` (Good), `E+2` (Evil)

### 씬 1: 길 위의 낯선 이

**배경**: 황혼 무렵, 좁은 산길에서 지친 노인이 무거운 짐을 지고 비틀거리고 있다. 당신은 목적지까지 한 시간이 남았고, 해가 지면 길이 위험해진다. 노인은 아무 말 없이 당신의 눈치를 보고 있다.

| 선택지 | 내용 |
|--------|------|
| **A** | 짐을 나눠 들고 함께 걷는다. 속도가 느려지더라도 혼자 두는 건 마음에 걸린다. |
| **B** | "짐을 여기 두고 내일 다시 오세요"라고 조언하고 혼자 서두른다. 지금은 내가 안전해야 한다. |

| **D** | 근처 마을에서 짐꾼을 구해주겠다고 약속하고 먼저 달려간다. 직접 돕는 것보다 실질적인 해결책이다. |

### 씬 7: 폐허 속 발견

**배경**: 오래된 던전을 탐색하던 중 아직 아무도 손대지 않은 방을 발견했다. 방에는 여러 갈래의 길이 있다. 함께 온 일행이 "어디로 갈지 네가 정해"라고 한다.

| 선택지 | 내용 |
|--------|------|
| **A** | 지도를 꺼내 현재 위치를 파악하고 가장 논리적인 루트를 분석한 뒤 결정한다. |

### 4.3 D&D 성향

```typescript
LC = L합산 - C합산
GE = G합산 - E합산

// L/C축
lawChaos = LC >= 4 ? "lawful" : LC <= -4 ? "chaotic" : "neutral"

// G/E축
goodEvil = GE >= 4 ? "good" : GE <= -4 ? "evil" : "neutral"

// 결합 (9가지 성향)
alignment = lawChaos + "-" + goodEvil
// 예: "lawful-good", "chaotic-neutral", "true-neutral"
```

**임계값 설계 근거**: G/E축 측정 씬 11개. 선(Good) 최대 달성 점수 +14, 악(Evil) 최대 달성 점수 −14으로 두 극단이 동등하게 도달 가능하도록 설계. ±4 임계값은 중립 구간(-3 ~ +3)이 혼합형 플레이를 자연스럽게 포용하면서 일관된 성향을 가진 플레이어는 명확하게 판정받도록 한다.
