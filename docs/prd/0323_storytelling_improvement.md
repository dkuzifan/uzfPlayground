# 스토리텔링 품질 개선 PRD (2026-03-23)

> 실제 플레이 테스트를 통해 발견된 문제들을 바탕으로,
> AI GM의 이야기 연출 능력과 NPC 등장 방식을 전면 개선합니다.

---

## 배경 — 무엇이 문제였나

실제로 게임을 플레이해본 결과, 다음과 같은 문제들이 발견됐습니다.

### "게임이 이야기처럼 느껴지지 않는다"

현재의 AI GM은 플레이어가 행동하면 반응하는 방식으로만 작동합니다.
플레이어가 아무 행동을 하지 않으면 아무 일도 일어나지 않고,
플레이어가 행동하면 AI가 그 행동에 맞춰 즉흥적으로 서사를 만들어냅니다.

이 방식의 문제는, **이야기에 흐름이 없다**는 것입니다.
소설이나 영화처럼 "도입 → 전개 → 절정 → 결말"의 감각이 없고,
매 턴이 동일한 무게를 갖는 단순한 반복처럼 느껴집니다.

### "NPC들이 너무 갑자기, 너무 많이 나온다"

실제 플레이 로그에서 확인된 문제입니다.

- 게임 시작 후 3턴 만에 NPC 4명이 전부 등장
- 탐색할 시간도, 세계관을 흡수할 시간도 없이 바로 갈등 상황
- 적 캐릭터가 소개도 없이 갑자기 나타나 대립
- 한 턴에 여러 NPC가 동시에 반응해서 읽기도 벅참

### "NPC 대사가 읽기 어렵다"

NPC의 상황 묘사(지문)와 실제 대사가 한 덩어리로 섞여 있어서,
어디서부터 실제 말인지 구분이 어렵습니다.

```
현재:
(지문: 비쩍 마른 오크가 어둠 속에서 모습을 드러낸다...) "대사: 무례하도다, 작은 그림자여."

개선 후:
[지문] 비쩍 마른 오크가 어둠 속에서 모습을 드러낸다...
💬 "무례하도다, 작은 그림자여."
```

---

## 개선 목표

> 플레이어가 한 편의 이야기를 경험하는 느낌을 받을 수 있도록.

1. AI GM이 이야기의 **설계자**로서 플롯을 미리 구성하고 단계적으로 이끈다
2. NPC는 **적절한 타이밍에 한 명씩** 극적으로 등장한다
3. 대사와 지문이 **시각적으로 분리**되어 읽기 쉬워진다

---

## 개선 항목 상세

---

### 1. Story Blueprint 시스템 (핵심 기능)

**한 줄 요약**: AI GM이 게임 시작 전 이야기의 4막 구조를 미리 설계한다.

#### 현재 방식

```
플레이어 행동 → GM이 즉흥 반응 → NPC 반응 → 반복
```

GM은 시나리오 설명만 가지고 있고, 이야기를 어디로 이끌어야 하는지 계획이 없습니다.

#### 개선 방식

```
게임 시작
    ↓
AI GM: 시나리오 + 목표 + 등장 NPC를 읽고
       "이 재료로 이야기를 어떻게 전개할 것인가"를 설계
    ↓
Story Blueprint (이야기 설계도) 생성:

  1막 [탐색]  : 어떤 단서를 발견하는가 / 어떤 분위기를 조성하는가
  2막 [긴장]  : 어떤 위협이 등장하는가 / 어떤 NPC와 처음 만나는가
  3막 [절정]  : 핵심 갈등이 무엇인가 / 어떻게 결판이 나는가
  4막 [해소]  : 성공/실패 후 어떤 결말로 마무리되는가
    ↓
매 턴: GM이 설계도를 참조하며 서사를 이끔
```

#### 플레이어가 느끼는 변화

- 처음 몇 턴은 세계관을 탐색하고 분위기에 익숙해지는 시간이 생긴다
- 이야기가 자연스럽게 긴장감을 높여가며 절정을 향해 나아간다
- "지금 이야기의 어느 지점에 있는가"를 감각적으로 느낄 수 있다

#### 기술 구현

- `Game_Session` 테이블에 `story_blueprint` 컬럼 추가 (JSONB)
- 게임 시작(`/sessions/[id]/start`) 시 Gemini가 블루프린트 생성
- 매 턴 GM에게 블루프린트를 컨텍스트로 전달
- 기존 시나리오에도 자동 적용 (게임 시작 시점에 생성하므로)

---

### 2. NPC 등장 타이밍 제어

**한 줄 요약**: NPC는 Story Blueprint에 따라 극적인 타이밍에 한 명씩 등장한다.

#### 현재 방식

NPC가 생성되는 순간 모두 "사용 가능" 상태가 되어,
게임 초반부터 여러 명이 동시에 등장합니다.

#### 개선 방식

NPC_Persona 테이블에 `is_introduced` (등장 여부) 상태를 추가합니다.

```
게임 시작: 모든 NPC → is_introduced = false (미등장 상태)

1막 탐색: NPC 등장 없음. 세계 탐색에 집중.
           (동료 NPC만 처음부터 함께)

2막 긴장: Blueprint의 지시에 따라 첫 번째 NPC 등장
           → GM 서사로 극적으로 소개
           → is_introduced = true로 변경

이후:    등장한 NPC만 대화/반응 가능
          새 NPC 등장은 Blueprint 단계에 따라 조율
```

#### 규칙

- 한 턴에 **새로 등장하는 NPC는 최대 1명**
- 첫 등장은 반드시 **GM 서사를 통해** 소개 (이름 없이 갑자기 대사 시작 금지)
- 탐색 단계(1막)에서는 적대적 NPC 등장 불가

---

### 3. 바이스탠더 반응 수 제한

**한 줄 요약**: 플레이어 행동 하나에 반응하는 NPC는 최대 2명으로 제한한다.

#### 현재 방식

플레이어가 행동하면 "이 행동에 반응할 NPC가 있는가?"를 모든 NPC에게 묻습니다.
NPC 4명이 있으면 최대 4명이 동시에 반응합니다.

#### 개선 방식

- 이미 등장한(`is_introduced = true`) NPC 중에서만 반응 가능
- 한 턴에 반응하는 NPC는 **최대 2명**으로 제한
- 가장 관련성 높은 NPC 순으로 우선 선택

---

### 4. GM 서사 품질 개선

**한 줄 요약**: GM이 이야기를 앞으로 밀어야 한다는 명확한 지시를 추가한다.

#### 현재 방식

GM은 플레이어 행동에 반응하는 데 집중합니다.
실패해도, 성공해도 상황이 크게 달라지지 않고 같은 패턴이 반복됩니다.

#### 개선 방식

GM 시스템 프롬프트에 다음 원칙을 추가합니다.

```
- 매 서사 후 상황이 최소 1가지는 반드시 변해야 한다
- 같은 정보를 두 번 반복하지 않는다
- 현재 막(1막~4막)의 톤과 긴장감을 유지한다
- Story Blueprint의 다음 단계로 자연스럽게 유도한다
```

---

### 5. NPC 대사 품질 개선

**한 줄 요약**: NPC가 짧고 강렬하게 말하도록, 반복을 금지한다.

#### 현재 방식

- NPC가 한 턴에 동일한 메시지를 담은 대사를 2~3번 반복
- 매 등장마다 같은 패턴의 말을 되풀이 ("~로다, 하찮은 자여!")
- 대사가 너무 길어서 임팩트가 없음

#### 개선 방식

NPC 시스템 프롬프트에 다음 제약을 추가합니다.

```
- 대사는 1~2문장으로 제한
- 이전 등장에서 한 말을 같은 맥락으로 반복하지 않는다
- 지문은 2~3문장으로 제한
```

---

### 6. NPC 지문/대사 분리 렌더링 (UI 개선)

**한 줄 요약**: 상황 묘사와 실제 대사를 시각적으로 분리해서 보여준다.

#### 현재 방식

NPC 응답이 하나의 텍스트 덩어리로 표시됩니다.

```
(지문: 그가 도끼를 번쩍 들며 으르렁댄다.) "대사: 네놈을 박살내겠다!"
```

#### 개선 방식

NPC 응답 구조를 분리합니다.

```json
{
  "stage_direction": "그가 도끼를 번쩍 들며 으르렁댄다.",
  "dialogue": "네놈을 박살내겠다!"
}
```

화면 표시:

```
[그롬쉬]
그가 도끼를 번쩍 들며 으르렁댄다.   ← 흐린 이탤릭체

  💬 "네놈을 박살내겠다!"           ← 말풍선 스타일
```

#### 향후 확장

구조가 분리되면, 나중에 NPC 캐릭터 이미지(바스트샷)를 추가하기도 쉬워집니다.

---

## 구현 우선순위

| 순위 | 항목 | 이유 |
|------|------|------|
| 1 | Story Blueprint | 근본 원인 해결. 다른 항목들의 기반 |
| 2 | NPC 등장 타이밍 + 바이스탠더 제한 | 가장 직접적으로 체감되는 문제 |
| 3 | GM 서사 + NPC 대사 개선 | 프롬프트 수정으로 비교적 빠르게 적용 |
| 4 | UI 지문/대사 분리 | 프론트엔드 작업. 별도로 진행 |

---

## 기존 시나리오 호환성

모든 개선 사항은 **기존에 저장된 시나리오에도 자동 적용**됩니다.

- Story Blueprint: 게임 시작 시 생성 → 시나리오 수정 불필요
- NPC 등장 제어: 세션 시작 시 초기화 → 기존 NPC에도 적용
- GM/NPC 프롬프트 개선: 코드 수정 → 즉시 전체 적용
- UI 분리: 프론트엔드 수정 → 즉시 전체 적용

---

## 기대 효과

```
개선 전:
행동 → 반응 → 행동 → 반응 (무한 반복, 목적 없음)

개선 후:
탐색 (세계 흡수) → 긴장 (첫 NPC 조우) → 절정 (핵심 갈등) → 해소 (결말)
```

플레이어가 "나 지금 이야기의 어느 부분을 경험하고 있구나"를 느낄 수 있게 됩니다.
