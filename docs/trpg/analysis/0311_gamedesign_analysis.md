# 게임 기획 분석 리포트 (2026-03-11)

> 게임 기획자/서비스 기획자 관점에서 현재 TRPG 프로젝트를 검토.
> 솔로 플레이와 멀티플레이 각각에서 재미있는 게임플레이를 위해 필요한 것을 분석.

---

## 현재 상태 진단

현재 구현된 것을 솔직하게 보면 **"AI가 반응하는 채팅방"** 에 가깝다.
TRPG라는 라벨이 붙어있지만, 플레이어가 기대하는 핵심 재미 요소들이 아직 많이 빠져있다.

---

## 솔로 플레이 분석

### 핵심 결여: 내 선택이 세계를 바꾸는 느낌이 없다

솔로 TRPG의 가장 강력한 동기는 **"내가 한 행동이 이야기를 실제로 바꿨다"** 는 확인이다.
현재는 행동 → GM 서술 → NPC 반응 → 다음 턴의 루프만 반복될 뿐이다.

---

### 문제 1: 스토리에 끝이 없다

게임이 언제 끝나는지 알 수 없다. 목표가 무엇인지도 불분명하다.
실제 TRPG는 **"용을 쓰러뜨린다"** 또는 **"마을에서 탈출한다"** 처럼 명확한 승리 조건이 있다.
끝이 없는 게임은 플레이의 동기를 잃게 만든다.

### 문제 2: 스토리 호흡이 없다

매 턴이 동일한 무게를 갖는다. 실제 TRPG는 다음과 같은 리듬이 있다.

```
평온한 탐색 → 긴장 고조 → 클라이맥스 → 해소
```

GM이 "지금은 탐색 구간", "지금은 전투", "지금은 결말"을 구분해서 연출하는 기능이 없다.

### 문제 3: 캐릭터 성장이 없다

세션 내에서 내 캐릭터가 강해지거나 변화하는 느낌이 없다.
HP가 깎이는 것만 있고, 무언가를 얻거나 배우거나 변하지 않는다.
아이템 획득, 스탯 변화, 새로운 능력 같은 보상 루프가 없으면 플레이 동기를 유지하기 어렵다.

### 문제 4: 발견의 드라마가 없다

TRPG의 큰 재미 중 하나는 숨겨진 것을 찾아내는 것이다.
`World_Dictionary`와 Lore 시스템이 구현되어 있음에도,
플레이어가 "뭔가를 알아냈다"는 경험을 받지 못한다.
발견이 드라마틱하게 제시되지 않아서이다.

### 문제 5: NPC가 먼저 행동하지 않는다

현재 NPC는 항상 플레이어의 행동을 받아서 반응한다.
살아있는 세계는 NPC가 먼저 무언가를 걸어오거나, 시간이 지나면 상황이 저절로 변해야 한다.
플레이어가 무언가를 하지 않으면 아무것도 일어나지 않는 세계는 죽은 세계처럼 느껴진다.

### 문제 6: 실패해도 괜찮은 설계가 없다

주사위 실패 시 스토리가 꺾이는데, 어느 방향으로 꺾이는지 GM 서술에만 의존한다.
"실패도 재미있다"는 경험 설계가 없다.
실패가 새로운 가능성을 열어주는 내러티브 분기가 되어야 한다.

---

## 멀티플레이 분석

### 핵심 결여: 같이 있을 이유가 없다

현재 멀티플레이는 **"순서를 기다리는 솔로 플레이 N개"** 다.

---

### 문제 1: 다른 플레이어의 턴이 나와 무관하다

다른 사람이 무엇을 하든 나는 그냥 기다린다. 구경꾼이다.
실제 TRPG 테이블에서는 다음과 같은 상호작용이 끊임없이 일어난다.

```
"잠깐, 내가 도와줄게"
"그건 나쁜 생각이야, 조심해"
"거기서 뒤를 조심해"
```

### 문제 2: 협력의 메커닉이 없다

두 명이 같은 NPC를 상대해도 각자의 행동이 서로에게 영향을 주지 않는다.
"내가 먼저 NPC의 주의를 끄는 동안 네가 문을 열어"라는 협력 플레이가 불가능하다.

### 문제 3: 플레이어 간 역할 분화가 없다

전사든 마법사든 게임에서 하는 행동이 동일하다.
직업이 선택지의 텍스트에만 영향을 줄 뿐,
"이 상황은 마법사만 해결할 수 있다"는 순간이 존재하지 않는다.

### 문제 4: 공동 결정 메커닉이 없다

"일행이 어디로 갈지 투표"하거나 "이 NPC를 믿을지 말지 논의"하는 구조가 없다.
각자 자기 턴에 자기 행동만 할 수 있다.

### 문제 5: 사회적 긴장감이 공유되지 않는다

멀티 TRPG의 큰 재미 중 하나는 플레이어 간의 드라마다.
누군가 주사위를 굴리는 순간을 다 같이 보며 느끼는 긴장감,
성공했을 때 함께 기뻐하는 것.
현재 UI에는 다른 플레이어의 주사위 롤 드라마가 공유되지 않는다.

---

## 개선 제안

### 🔴 1순위 — 스토리에 끝을 만들어야 한다 (엔딩 시스템)

**시나리오 승리/실패 조건 시스템**

시나리오에 `victory_conditions`와 `defeat_conditions`를 설정하고,
조건 충족 시 GM이 엔딩 서술을 생성하며 게임이 종료된다.

```
승리 조건 예시
- "용의 심장을 획득한다"
- "마을 주민 중 5명 이상이 생존한 채 탈출한다"
- "진범의 이름을 GM에게 선언한다"

패배 조건 예시
- "파티 전원 HP 0"
- "특정 NPC가 사망"
- "30턴 내 탈출 실패"
```

이것 하나만 있어도 플레이의 목적 의식이 생긴다.

---

### 🔴 2순위 — 다른 플레이어의 턴도 나와 관련 있어야 한다

**관전자 반응 시스템**

다른 플레이어의 턴에 간단한 반응 버튼을 제공한다.

```
😨 긴장돼   🎉 잘했어   ⚠️ 조심해   👀 흥미롭네
```

기다리는 플레이어가 완전한 구경꾼이 아니게 된다.

**보조 행동 시스템**

다른 플레이어 턴 중 "지원 선언"을 할 수 있다.
이것이 현재 행동 플레이어의 주사위 판정에 보너스를 준다.

```
예: "내가 마법으로 빛을 밝혀줄게" → 현재 턴 플레이어 주사위 +2
    "내가 옆에서 위협할게"        → 설득 판정 DC -3
```

---

### 🟡 3순위 — NPC가 먼저 말을 걸어야 한다 (NPC 이벤트 트리거)

**조건 기반 NPC 자발 행동**

특정 조건이 충족되면 플레이어 행동 없이도 NPC가 먼저 행동한다.

```
조건 예시
- 공포 수치 ≥ 80 → NPC가 도망치거나 항복 선언
- 호감도 ≥ 90    → NPC가 비밀 정보를 먼저 공유
- N턴 동안 상호작용 없음 → NPC가 먼저 말을 걸어옴
- 특정 Lore 키워드가 최근 로그에 등장 → 관련 NPC가 반응
```

---

### 🟡 4순위 — 발견의 드라마를 만들어야 한다 (로어 공개 연출)

**발견 카드 UI**

Lore가 트리거됐을 때 채팅 로그에 별도 형식의 "발견" 카드를 표시한다.
단순히 NPC 대사에 녹아드는 방식에서 벗어나, 시각적으로 강조한다.

```
╔══════════════════════════╗
║  📜 새로운 단서 발견       ║
║                          ║
║  "어둠의 제단은            ║
║   달이 없는 밤에 열린다..." ║
╚══════════════════════════╝
```

플레이어가 무언가를 알아냈다는 성취감을 명확하게 전달한다.

---

### 🟢 5순위 — 게임 내 간단한 성장 (소지품 시스템)

**아이템/메모 목록**

아이템 시스템까지는 아니어도,
"이 게임에서 내가 획득한 것"을 기록하는 소지품 목록을 제공한다.

```
플레이어 캐릭터 카드에 표시:
보유 아이템: 낡은 지도, 신비한 보석 1개, 열쇠 묶음
```

GM이 중요한 물건을 줄 때 이 목록에 자동 추가된다.
세션이 끝났을 때 "이번 모험에서 얻은 것"을 시각적으로 확인할 수 있다.

| 관점 | 핵심 문제 | 최우선 개선 |
|------|-----------|------------|

| 멀티 | 같이 있을 이유가 없다. 기다리는 게 전부다 | 보조 행동 + 반응 시스템 |
