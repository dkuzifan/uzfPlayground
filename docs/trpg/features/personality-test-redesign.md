## 2. 설계 원칙

### 원칙 A: 각 씬은 축 중립적이어야 한다 (MBTI / D&D)

각 씬에서 4개 선택지의 축 점수 합 = 0:

```
Σ(ei) = 0,  Σ(sn) = 0,  Σ(tf) = 0,  Σ(jp) = 0
Σ(lc) = 0,  Σ(ge) = 0
```

이 제약을 만족하면 랜덤 선택 시 기댓값 = 0 → 모든 결과 동등 확률.

권장 점수 패턴 (합 = 0):
- 강: `{+2, +1, -1, -2}`
- 중: `{+1, +1, -1, -1}`
- 분산: `{+2, 0, 0, -2}` / `{+1, 0, 0, -1}`
- 비관여: `{0, 0, 0, 0}` (해당 축을 이 씬에서 측정하지 않음)

### 원칙 B: 에니어그램은 타입별 동등 할당

- 18씬 × 4선택지 = **72 choice-slots**
- 72 / 9 types = **타입당 정확히 8 slots**
- 각 선택지는 **1개의 주 타입(3pt)** + 필요 시 **1개의 부 타입(1pt)** 부여
- 타입당 총 획득 가능 포인트 동일하게 설계 (주 3pt × 8 = 24pt 기준)

## 5. 씬별 스코어 설계 규칙

### 선택지 구조 (예시: 씬 3 — 주 포커스 TF + LC, 에니어그램 타입 1, 3)

```
A: tf:+2, lc:+2, ei:0, sn:0, jp:0, ge:0, enn:[[1,3]]
B: tf:+1, lc:-1, ei:0, sn:0, jp:0, ge:0, enn:[[3,3]]
C: tf:-1, lc:+1, ei:0, sn:0, jp:0, ge:0, enn:[[1,1],[3,1]]
D: tf:-2, lc:-2, ei:0, sn:0, jp:0, ge:0, enn:[[3,1],[1,1]]
합계: tf=0 ✓, lc=0 ✓
```

### 비관여 축 처리
- 원칙적으로 0으로 설정
- 이야기 자연스러움을 위해 소폭 부여 시 반드시 합 = 0 유지
- 예: ei를 소폭 반영하려면 `{+1, -1, +1, -1}` 또는 `{+1, +1, -1, -1}` 패턴 사용

## 6. 판정 알고리즘

기존 알고리즘 구조 유지:
- **MBTI**: 축별 총합 양수/음수/0(타이브레이커)
- **에니어그램**: 타입별 총점 최댓값
- **D&D**: LC 총합, GE 총합 → 구간 판정 (±4 기준)

단, 씬 수 변경(12→18)에 따라 타이브레이커에서 참조하는 씬 인덱스 재검토 필요.
편향이 제거되면 타이브레이커 발동 빈도 자연스럽게 감소할 것.
