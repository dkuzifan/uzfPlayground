## Context

### 현재 구조

`hit-result.ts`의 타구 판정은 현재 **이분법(binary)**:

```
roll < p_out  →  아웃 (포구 성공)
roll ≥ p_out  →  안타 (포구 실패 = 모두 히트)
```

실제 야구에서는 포구 실패가 **두 가지**로 나뉜다:
- **안타(Hit)**: 수비수 도달 불가 → 타자 능력으로 친 것
- **실책(Error)**: 수비수 도달 가능했지만 실수 → 타자 기록 없음, 수비수 E 기록

### 현재 문제

**문제 1 — 포구 실책(Fielding Error)이 없다**
`p_out`이 높은 공(잡을 수 있는 공)을 놓쳐도 전부 안타로 처리된다.

**문제 2 — 송구 실책(Throwing Error)이 없다**
`resolveThrow`가 safe/out 이분법이라 폭투로 인한 추가 진루가 발생하지 않는다.

**문제 3 — 자책점(ER) 구분이 없다**
`PitcherGameStats.ER` 주석: "수비 에러 미구현 → 모든 실점을 자책점으로 처리". 실책 후 실점은 비자책점(R만 증가, ER은 증가 안 함)이어야 한다.

**문제 4 — 수비수 E(에러) 스탯이 없다**
실책을 범한 수비수의 기록이 남지 않는다.

## Goals / Non-Goals

### Goals (Must-have)

**G1. 포구 실책 — `resolveHitResult`**

잡을 수 있는 공을 놓치면 안타가 아닌 실책으로 판정.

```
roll < p_out              → 아웃 (포구 성공)
p_out ≤ roll < p_out + p_error → 실책 (잡을 수 있었지만 실수)
roll ≥ p_out + p_error    → 안타 (수비 범위 밖)

p_error = p_out * ERROR_COEFF   // 실책 계수: ~0.04
```

`p_out`이 높을수록 `p_error`도 커짐 — "충분히 잡을 수 있는 공일수록 실책 가능성 존재"

**G2. `AtBatResult`에 `'reach_on_error'` 추가**

실책 출루 = 타수 기록, 안타 기록 없음, 출루 가능.

**G3. 송구 실책 — `resolveThrow` 확장**

송구 판정에 폭투 확률 추가. 실책 시 주자 +1 추가 진루 (항상 +1 베이스, 멀티베이스 제외).

```
p_throw_error = f(throw_dist, arm_stat, pressure)
// pressure: 아웃 직전 margin이 작을수록 높음
```

폭투 발생 시: `verdict = 'wild_throw'` → 주자 현재 목표 베이스에서 +1

**G4. 수비수 E 스탯 — `FielderGameStats`**

```typescript
interface FielderGameStats {
  player: Player
  E: number  // 실책 수
}
```

팀별 `fielders: FielderGameStats[]` 집계. `calc-game-stats.ts`에서 `error` 이벤트 처리.

**G5. 자책점 구분 — `R` vs `ER`**

실책 이후 출루한 주자의 득점 = 비자책점.
- `PitcherGameStats.ER` → 실책 없이 허용한 실점만 누적
- `PitcherGameStats.R` 필드 추가 (총 실점)
- 실책 주자 추적: `error_runners` Set (주자 id 기반)

### Non-Goals

| 항목 | 이유 |
|------|------|
| 포수 방해(Catcher's Interference) | 별도 규칙, 범위 초과 |
| 타격 방해 / 주루 방해 | 별도 피처 |
| 송구 실책 멀티베이스 진루 | MVP에서 +1로 단순화. 목표 베이스에서 한 칸 더, 방향 무관 동일 적용 |

## Success Definition

- 잡을 수 있는 공이 경기당 평균 0~2개 실책으로 이어진다 (과도하지 않은 빈도)
- 실책 출루 주자가 득점해도 투수 ER은 증가하지 않는다
- 폭투로 주자가 추가 진루하는 상황이 발생한다
- 실책을 범한 수비수에게 E 스탯이 기록된다
- 삼진/볼넷/안타/홈런/병살/태그업 등 기존 플로우 정상 동작 유지
