## Context

셋업 플로우(`/arena/baseball/setup`)에서 "경기 시작"을 누르면 `/arena/baseball/game`으로
이동하지만, 현재 게임 화면은 설정 요약만 표시하는 플레이스홀더다.

야구 시뮬레이터 엔진(`runGame`)은 이미 완성되어 있으며 전체 경기를 **한 번에 배치로 실행**하여
`GameEvent[]`를 반환한다. 게임 화면은 이 이벤트 배열을 순서대로 재생하여
마치 경기가 실시간으로 진행되는 것처럼 표시하는 **이벤트 플레이백 UI**다.

### 벤치마크

**MLB 공식 앱의 Gameday 화면**을 디자인 기준으로 삼는다. (실제 화면 확인 완료)

확인된 화면 구조:
- **상단 배너**: 팀 로고 + 양팀 점수 + 현재 이닝 (예: `BOT 3`)
- **좌측 패널 (메인)**: 투구별 play-by-play 리스트
  - 각 항목: 컬러 원(볼=파랑, 스트라이크=빨강, 인플레이=초록) + 구종/구속 + 카운트
  - 타석 결과(홈런·안타 등)는 굵게 강조 + 플레이 설명 텍스트
  - 다이아몬드(소형)로 주자 위치 표시
- **중앙**: 3D 타자 영상 + 스트라이크존 투구 번호 오버레이
- **우측 패널(데스크탑)**: 박스스코어 전체 테이블
- **하단**: 현재 투수 정보(구수·이닝·탈삼진) + 타자 매치업(On deck / In the hole)
- **탭**: Live / Summary / Box

**우리 화면과의 차이**: 실제 영상(3D/중계)이 없으므로 우측 패널을 다이아몬드 + 볼카운트 + **스트라이크 존** 패널로 대체한다. 스트라이크 존은 현재 타석의 투구 위치를 번호 도트로 표시한다 (볼=파랑, 스트라이크=빨강, 파울=노랑). 나머지 정보 구조(배너·play-by-play·하단 매치업·탭)는 동일하게 적용한다.

디자인 언어(색상·타이포)는 본 프로젝트의 다크모드 스타일(Tailwind, 기존 `/arena` 계열)에 맞게 적용한다.

### 이벤트 플레이백 모델

```
runGame() 실행 → GameEvent[] 전체 생성
         ↓
진행 단위에 따라 이벤트 그룹화
  - 타석 기준: at_bat_result 단위로 묶음
  - 투구 기준: pitch 이벤트 하나씩
         ↓
자동(속도 조절) 또는 수동("다음" 버튼)으로 한 그룹씩 공개
```

### 게임 모드 처리 (MVP)

- **풀 시뮬레이션**: 자동 플레이백 (속도 조절 가능) + "다음" 수동 진행 지원
- **감독 모드**: MVP에서는 풀 시뮬레이션과 동일하게 동작 (수동 결정 UI는 별도 피처)

## Goals / Non-Goals

**Goals (MVP):**
- **G1** 경기 시작 시 `runGame()` 실행 → `GameEvent[]` 전체 생성
- **G2** 스코어보드 — 팀명, 현재 이닝(초/말), 현재 점수 실시간 갱신
- **G3** 다이아몬드 — 주자 위치(1·2·3루) 시각화, 아웃 카운트 표시
- **G4** 현재 타석 정보 — 현재 타자·투수 이름, 볼카운트(B-S-O)
- **G5** 이벤트 로그 — 투구/타석 결과를 한국어 문자 중계 텍스트로 표시 (스크롤)
- **G6** 진행 단위 선택 반영 — 타석 기준(at_bat 단위 공개) / 투구 기준(pitch 단위 공개)
- **G7** 속도 조절 — 자동 플레이 시 빠름/보통/느림 선택 + 일시정지
- **G8** "다음" 수동 진행 버튼 (일시정지 상태에서 한 단위씩 수동 진행)
- **G9** 경기 종료 후 결과 화면 — 최종 점수, 라인스코어, 박스스코어(타자/투수)
- **G10** 결과 화면에서 "다시 하기"(셋업으로) / "타이틀로" 버튼

**Non-Goals:**
- 감독 모드 수동 결정 UI (구종 선택, 번트 지시 등) — 별도 피처
- 경기 저장/불러오기 — 별도 피처
- 소리/효과음 — 별도 피처

## UX Acceptance Criteria

**UC1. 게임 진입**
- `/arena/baseball/game` 접근 시 자동으로 경기 시뮬레이션 실행 후 플레이백 시작
- GameConfig 없으면 즉시 `/arena/baseball/setup`으로 redirect

**UC2. Live 탭 — 플레이백**
- 이벤트 공개 시마다 배너(점수·이닝), 다이아몬드(주자·아웃), 매치업 바가 즉시 갱신
- 새 항목 추가 시 로그가 자동으로 최하단 스크롤
- 이닝 변경 시 구분선이 새 항목 위에 삽입됨

**UC3. 컨트롤**
- 일시정지 상태에서 "다음" 버튼 클릭 시 정확히 한 단위만 진행
- 속도 변경은 즉시 적용 (현재 interval에 반영)
- 경기 종료 후 컨트롤 영역이 사라지고 결과 화면으로 전환

**UC4. Box 탭**
- 탭 전환 시 현재까지의 누적 스탯 표시
- 경기 진행 중에도 Box 탭 조회 가능

**UC5. 결과 화면**
- `game_end` 이벤트 감지 후 자동 전환
- 끝내기(`walk_off`) 시 별도 배지 표시

## User Flow

```mermaid
flowchart TD
  A[/arena/baseball/game 진입] --> B{GameConfig?}
  B -->|없음| C[/arena/baseball/setup redirect]
  B -->|있음| D[runGame 실행 → events 생성]
  D --> E[Live 탭 자동 플레이백 시작]
  E --> F{game_end 이벤트?}
  F -->|아직| G[이벤트 순서대로 공개]
  G --> E
  F -->|종료| H[결과 화면]
  H -->|다시 하기| C
  H -->|타이틀로| I[/arena/baseball]
```

## Requirements

### Must-have

**R1. 게임 실행**
- 페이지 마운트 시 localStorage에서 `GameConfig` 로드
- `getTeamById()`로 홈/원정 팀 구성 후 `runGame()` 실행 → `GameEvent[]` 전체 생성
- 설정 없으면 `/arena/baseball/setup`으로 redirect

**R2. 상단 배너**
- 양팀 이름 + 현재 점수 + 현재 이닝(초/말, 예: `BOT 3`) 표시
- 이벤트 진행에 따라 점수·이닝 실시간 갱신

**R3. 다이아몬드 + 스트라이크 존 패널**
- 1·2·3루 주자 유무 시각화 (주자 있는 루 강조)
- 아웃 카운트 표시 (● ● ○ 형태)
- 볼카운트 표시 (B · S)
- 스트라이크 존: 3×3 내부 그리드 + 현재 타석의 투구 위치 번호 도트 (볼=파랑, 스트라이크=빨강, 파울=노랑, 인플레이=초록) — 타석 변경 시 초기화

**R4. Play-by-Play 로그 (Live 탭)**
- 각 투구 항목: 컬러 원(볼=파랑, 스트라이크=빨강, 인플레이=초록, 파울=노랑) + 결과 텍스트
- 타석 결과(안타·홈런·삼진 등) 강조 표시 + 한국어 플레이 설명
- 이닝 변경 시 구분선 + 이닝 레이블 (예: `▸ 3회 말`)
- 새 이벤트 추가 시 로그 최하단으로 자동 스크롤
- 투수 교체 이벤트 별도 표시

**R5. 하단 매치업 바**
- 현재 투수: 이름, 구수, IP, 탈삼진
- 현재 타자: 이름, 현재 타석 결과 (타석 진행 중이면 현재 카운트)
- On deck 타자 이름

**R6. 탭 구조**
- **Live**: Play-by-Play 로그 + 다이아몬드 + 매치업
- **Box**: 박스스코어 (양팀 타자 AB·H·HR·RBI·AVG, 투수 IP·H·ER·BB·SO·ERA·결과)

**R7. 진행 단위 반영**
- 타석 기준: `at_bat_result` 단위로 묶어서 한 번에 공개 (그 타석의 투구들 + 결과)
- 투구 기준: `pitch` 이벤트 하나씩 순서대로 공개

**R8. 플레이백 컨트롤**
- 자동 재생: 빠름(0.5s) / 보통(1.5s) / 느림(3s) 속도 선택
- 일시정지 / 재생 토글 버튼
- "다음" 수동 진행 버튼 (일시정지 상태에서 한 단위씩)

**R9. 경기 종료 결과 화면**
- 경기 종료(`game_end` 이벤트) 감지 시 결과 화면으로 전환
- 최종 점수 + 승자 표시 (끝내기 여부 포함)
- 라인스코어 (이닝별 득점)
- 박스스코어 (타자·투수)
- "다시 하기" → `/arena/baseball/setup` / "타이틀로" → `/arena/baseball`

### Nice-to-have

**N1. 득점 장면 강조**
- `score` 이벤트 발생 시 배너 점수 영역 잠깐 강조(flash)

**N2. 끝내기 특별 표시**
- `reason: 'walk_off'`인 경우 결과 화면에 "끝내기 승리" 강조 배지

## Success Definition

- 셋업에서 "경기 시작" → 게임 화면 진입 시 경기가 자동으로 시작됨
- 이벤트 로그에 경기 흐름이 한국어 문자 중계 형태로 표시됨
- 스코어보드·다이아몬드·볼카운트가 이벤트에 따라 올바르게 갱신됨
- 속도 조절 및 일시정지/수동 진행이 동작함
- 경기 종료 후 결과 화면에서 라인스코어·박스스코어 확인 가능
- 셋업으로 돌아가기 및 타이틀로 이동 가능
