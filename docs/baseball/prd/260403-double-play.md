## Context

### 현재 문제

**문제 1 — 내야 그라운더 아웃 시 주자들이 이동하지 않는다**

`advanceRunners('out', ...)`는 `fixedAdvance`를 거쳐 아무 주자도 이동시키지 않는다.
포스 상황의 주자는 타자가 아웃이더라도 **반드시 다음 베이스로 이동**해야 한다.

```
현재: 1루 주자 있음 + 내야 그라운더 아웃 → 1루 주자 그대로 (버그)
기대: 1루 주자 → 2루 (포스 진루, 아웃 처리)
```

**문제 2 — 병살 판정이 없다**

포스아웃 후 피벗 맨이 1루로 추가 송구하는 병살 플레이가 없다.

**문제 3 — 비포스 주자의 홈 도전 판단이 없다**

3루 주자는 내야 그라운더 상황에서 홈을 노릴 수 있으나, 현재 아무 판단도 없다.

## Goals / Non-Goals

### Goals (Must-have)

---

**G1. 포스 아웃 전면 구현 — 모든 포스 케이스 처리**

포스 상황 = "타자가 1루를 차지함으로써 후속 주자가 다음 베이스로 밀려나는 상황".
이 경우 해당 주자는 **판단 없이 무조건 다음 베이스로 진루** (그리고 내야수 송구로 아웃).

포스가 발생하는 베이스별 케이스:

| 주자 상황 | 포스 베이스 |
|----------|-----------|
| 1루만     | 2루 (1루 주자 → 2루) |
| 1·2루    | 2루(1루→2루) + 3루(2루→3루) |
| 만루      | 2루 + 3루 + 홈(3루→홈) |

모든 포스 주자는 아웃 처리 (포스 베이스로 공이 도달 = 즉시 아웃, 타이밍 판정 없음).

---

**G2. 병살 판정 — 피벗 송구 vs 타자 도달 시간 비교**

포스아웃 직후 그 베이스의 수비수(피벗 맨)가 1루로 추가 송구.
타자의 1루 도달 시간 vs 피벗 송구 시간을 비교해 병살 성공/실패 결정.

```
pivot_throw_time = relay_reaction(0.3s) + dist(포스 베이스 → 1루) / pivot_throw_speed
t_batter_to_1B  = 27.43m / batter_run_speed

if t_batter_to_1B > pivot_throw_time  → 병살 성공 (2 outs_added)
else                                   → 포스아웃만 (1 outs_added, 타자 1루 세이프)
```

포스 베이스별 피벗 경로:

| 포스 베이스 | 피벗 맨 | 추가 송구 경로 |
|-----------|--------|-------------|
| 2루       | SS / 2B | 2루 → 1루 (6-4-3 / 4-6-3) |
| 3루       | 3B     | 3루 → 1루 (5-3) |
| 홈        | C      | 홈 → 1루 (2-3) |

내야수가 여러 포스 베이스 중 어디로 던질지는 **포스 베이스 우선 순위**로 결정:
- 병살에 가장 유리한 베이스(피벗 후 1루 도달이 빠른 쪽) 우선.
- 단순화: 2루가 포스라면 2루 우선. 만루라면 홈 포스가 기본 선택.

---

**G3. 3루 주자 홈 도전 판단**

3루 주자는 내야 그라운더에서 홈을 노릴 수 있다. 다음 기준으로 판단:

```
Case A: 아웃 위험이 낮다고 판단할 때 (decideChallengeAdvance → true)
Case B: 2아웃 상황 — 이닝이 끝날 수도 있으므로 무조건 홈 도전
```

`decideChallengeAdvance` 로직은 기존 #3-b 것을 그대로 활용.
내야 그라운더는 공이 빠르게 잡히므로 일반적으로 마진이 음수 → 대부분 도전 포기.
(슬로우 그라운더나 내야수 위치에 따라 간헐적으로 도전 발생)

**2루 주자 홈 도전 없음**: 내야 그라운더에서 2루 → 홈까지 거리가 너무 멀어 현실적으로 불가. 실책 없이는 시도하지 않음. 이번 MVP에서는 구현하지 않음.

---

**G4. 야수 선택(Fielder's Choice) 규칙 — 설정 옵션**

타자가 내야 그라운더를 쳤을 때 수비수가 1루가 아닌 다른 베이스로 송구하고 타자가 1루에 세이프되는 상황.

두 가지 기준이 존재하며, 게임 설정에서 선택 가능:

| 옵션 | 이름 | 야수 선택 판정 조건 |
|------|------|-----------------|
| `'mlb'` | MLB 기준 | 타자가 1루 세이프이면 무조건 야수 선택 (선행 주자 아웃 여부 무관) |
| `'standard'` | 일반 기준 | 선행 주자를 잡으려 했는데 그 주자까지 세이프된 경우에만 야수 선택 |

기본값: `'mlb'`

이 설정은 스탯 기록(안타/야수선택 분류)에 영향을 줌.

---

**G5. 이벤트 로깅**

| 결과 | `at_bat_result` 값 | 설명 |
|------|-------------------|------|
| 병살 성공 | `'double_play'` | 포스아웃 + 타자 1루 아웃, 2 outs |
| 포스아웃 + 타자 세이프 | `'fielders_choice'` | 포스아웃만, 타자 1루 진루 |
| 포스 없음 (주자 없음/비포스) | 기존 `'out'` | 타자만 아웃 |

| 항목 | 처리 |
|------|------|
| 2루 주자 홈 도전 (내야 그라운더) | → #6 에러에서 처리 (실책 발생 시 추가 진루 연계) |

## Success Definition

- 1루 주자 있는 내야 그라운더 아웃 시, 1루 주자가 반드시 2루로 이동(아웃)된다
- 만루 내야 그라운더 시, 전 주자가 한 베이스씩 강제 진루(아웃)된다
- 피벗 맨의 Throw 스탯이 높을수록 병살 성공률이 높다
- 타자의 Running 스탯이 높을수록 병살을 피할 확률이 높다
- 3루 주자가 2아웃 시 홈으로 달린다
- 야수 선택 기준(mlb/standard)이 스탯 분류에 올바르게 반영된다
- 포스 주자 없는 내야 그라운더, 외야 아웃, 삼진 등 기존 결과에 영향 없음
