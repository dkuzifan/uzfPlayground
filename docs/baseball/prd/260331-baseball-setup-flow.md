## Context

`/arena/baseball`에서 "한 경기 플레이"를 클릭하면 `/arena/baseball/setup`으로 이동하지만
현재는 플레이스홀더 페이지만 존재한다.

게임을 시작하려면 다음 정보를 사용자에게 수집해야 한다:
1. **내 팀** — 어떤 팀을 선택할지
2. **상대 팀** — 상대방 팀 선택
3. **구장** — 경기를 치를 구장 선택
4. **홈/원정** — 내 팀의 홈/원정 여부
5. **게임 모드** — 감독 모드(수동) vs 풀 시뮬레이션(자동)
6. **진행 단위** — 타석 기준 vs 투구 기준

팀 데이터는 이번 피처에서 정적 파일(`src/lib/baseball/data/teams.ts`)로 하드코딩한다.
DB 연동은 별도 피처(시즌 모드 / 커스텀 팀)에서 처리한다.

### 셋업 플로우 단계

```
/arena/baseball/setup
  ├─ Step 1: 팀 선택    — 내 팀 / 상대 팀 선택 (캐러셀 또는 그리드)
  ├─ Step 2: 경기 설정  — 구장 선택, 홈/원정, 게임 모드, 진행 단위
  └─ Step 3: 프리게임   — 로스터 확인 + 게임 시작
```

---

## Goals / Non-Goals

**Goals (MVP):**
- **G1** 정적 팀 데이터 파일 생성 — 플레이 가능한 팀 3~5개 (각 팀: 이름, 색상, 선발 투수 1 + 타자 9 + 불펜 2)
- **G2** Step 1: 팀 선택 화면 — 한 화면에서 내 팀 + 상대 팀 동시 선택 (양쪽 그리드)
- **G3** Step 2: 경기 설정 화면 — 구장 선택, 홈/원정, 게임 모드(감독/풀 시뮬레이션), 진행 단위(타석/투구)
- **G4** Step 3: 프리게임 화면 — 선택한 팀 로스터 열람(읽기 전용) + "경기 시작" 버튼
- **G5** 셋업 완료 시 `/arena/baseball/game`으로 이동 (게임 설정을 state/localStorage로 전달)
- **G6** 각 단계에서 이전 단계로 돌아가기 가능

**Non-Goals:**
- 커스텀 팀 생성/편집 — 별도 피처
- DB 연동 팀 데이터 — 시즌 모드 피처에서 처리
- 로스터 편집 (타순 변경, 선수 교체, 불펜 운용) — 별도 피처로 기획/구현
- 실제 게임 화면 구현 — 별도 피처

## UX Acceptance Criteria

**UC1. Step 1 — 팀 선택**
- 화면 진입 시 양쪽 모두 미선택 상태, "다음" 비활성
- 한쪽에서 팀 선택 시 해당 팀의 primary_color로 카드 강조
- 한쪽에서 이미 선택된 팀을 반대쪽에서 선택하면 선택 불가 (카드 dim 처리)
- 양쪽 모두 선택 완료 시 "다음" 활성화

**UC2. Step 2 — 경기 설정**
- 구장, 홈/원정, 게임 모드, 진행 단위 4개 항목 모두 선택되어야 "다음" 활성화
- 기본값 없음 (사용자가 직접 모두 선택해야 함)

**UC3. Step 3 — 프리게임**
- 선택한 설정 요약이 상단에 표시됨
- "경기 시작" 클릭 시 설정을 localStorage에 저장 후 `/arena/baseball/game`으로 이동

**UC4. 공통 네비게이션**
- 상단 스텝 인디케이터에서 현재 단계가 강조됨
- "이전" 버튼으로 이전 단계 복귀 시 이전 선택값 유지

## Requirements

### Must-have

**R1. 정적 팀 데이터**
- `src/lib/baseball/data/teams.ts` — 플레이 가능한 팀 4개 하드코딩
- 각 팀: `id`, `name`, `short_name`, `primary_color`, 선발 투수 1명 + 타자 9명(포수 포함) + 불펜 2명
- 기존 `simulate-game.mjs`의 팩토리 함수 패턴을 참고하여 현실적인 스탯으로 구성

**R2. 정적 구장 데이터**
- `src/lib/baseball/data/stadiums.ts` — 구장 3~4개 하드코딩
- 각 구장: `id`, `name`, `location` (도시명)
- MVP에서 구장은 표시/선택 용도 (경기 로직에 영향 없음)

**R3. Step 1 — 팀 선택**
- 한 화면에 "내 팀" / "상대 팀" 두 영역을 나란히 표시
- 각 영역에서 팀 목록(카드 그리드)을 보여주고 하나를 선택
- 같은 팀을 양쪽에 선택할 수 없음 (중복 방지)
- 양쪽 모두 선택되어야 "다음" 버튼 활성화

**R4. Step 2 — 경기 설정**
- **구장 선택**: 구장 목록에서 하나 선택 (카드 또는 라디오 형태)
- **홈/원정**: 내 팀이 홈인지 원정인지 선택 (토글 또는 버튼 2개)
- **게임 모드**: 감독 모드 / 풀 시뮬레이션 선택
- **진행 단위**: 타석 기준 / 투구 기준 선택
- 모든 항목 선택 시 "다음" 버튼 활성화

**R5. Step 3 — 프리게임**
- 선택한 두 팀의 로스터 표시 (선발 투수 + 타자 9명 + 불펜)
- 선택한 구장, 홈/원정, 게임 모드 요약 표시
- "경기 시작" 버튼 → `/arena/baseball/game`으로 이동
- 게임 설정(팀, 구장, 홈/원정, 게임 모드, 진행 단위)을 `localStorage`에 저장하여 전달

**R6. 단계 간 네비게이션**
- 각 단계 상단에 진행 표시 (Step 1 / 2 / 3)
- "이전" 버튼으로 이전 단계로 복귀 가능
- Step 1에서 "이전" → `/arena/baseball`로 이동

**R7. 팀 색상 반영**
- 선택된 팀 카드에 해당 팀의 `primary_color`로 테두리/배경 강조

### Nice-to-have

**N1. 구장 설명**
- 구장 카드에 간단한 설명 또는 수용 인원 표시

## Success Definition

- `/arena/baseball`에서 "한 경기 플레이" 클릭 → Step 1(팀 선택)로 진입
- 3단계를 순서대로 완료 후 "경기 시작" 클릭 → `/arena/baseball/game`으로 이동
- 각 단계에서 뒤로가기 동작
- 선택한 게임 설정이 게임 화면으로 올바르게 전달됨
