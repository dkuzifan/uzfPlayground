## Context

현재 게임 화면(`/arena/baseball/game`)의 `ZoneFooter` 컴포넌트에는 64×52px짜리 작은 다이아몬드가 있다.
이 다이아몬드는 각 루에 주자가 "있음/없음" 여부만 정적으로 표시한다.

사용자는 주자가 루를 이동할 때 **바둑알 형태의 도트가 루 경로를 따라 실시간으로 이동**하는 시각적 피드백을
원한다. 단순한 위치 표시가 아닌, 이동 동작 자체가 화면에 보이는 것이 핵심이다.

### 데이터 모델 현황

- `runner_advance` 이벤트: `moves: Array<{ runner: Player; from: 1|2|3|'batter'; to: 1|2|3|'home' }>`
- `steal_attempt` 이벤트: `{ runner: Player; from: 1|2 }`
- `steal_result` 이벤트: `{ runner: Player; from: 1|2; to: 2|3|'home'; success: boolean }`
- `tag_up` 이벤트: `{ runner: Player; from: 1|2|3; to: 1|2|3|'home'; safe: boolean }`
- 이미 from/to 데이터가 있으므로 엔진 변경 없이 UI 레이어에서 애니메이션 구현 가능

### 구현 방식

배치 실행된 `GameEvent[]`를 순서대로 재생하는 플레이백 구조이므로,
**엔진에서 물리 좌표를 실시간 생성하는 것은 불필요**하다.
이벤트 발생 시점에 from → to 경로를 CSS transition으로 보간하는 것으로 충분하다.
(MLB Gameday 앱도 동일 방식 사용)

## Goals / Non-Goals

**Goals (MVP):**
- **G1** 기존 소형 다이아몬드를 더 큰 독립 컴포넌트(`RunnerDiamond`)로 교체
- **G2** 주자 도트가 `runner_advance` 이벤트 발생 시 from → to 경로를 따라 CSS transition으로 이동
- **G3** 도루(`steal_attempt` → `steal_result`) 이벤트에도 동일한 애니메이션 적용
- **G4** 태그업(`tag_up`) 이벤트에도 동일한 애니메이션 적용
- **G5** 홈런/만루 등 여러 주자가 동시에 이동할 때 각 도트가 독립적으로 이동
- **G6** 다이아몬드가 게임 화면 레이아웃 내에서 충분히 크게 표시됨

**Non-Goals:**
- 물리 기반 중간 좌표 생성 (엔진 수정 없음)
- 수비수 도트 표시
- 소리/진동 효과
- 투구 중 주자 리드 표현 (이벤트 데이터 없음)

### 엣지 케이스 처리 방침

- **아웃 도트**: 아웃 종류에 따라 처리 방식이 다르다.
  - 도루 아웃: 주자가 실제로 목표 베이스까지 뛰었으므로 목표 베이스에 도달한 후 페이드아웃
  - 태그업 아웃·송구 아웃: 아웃 이벤트 발생 즉시 페이드아웃
- **동시 이동**: 여러 주자가 같은 이닝에 동시에 이동할 때 각 도트는 독립적으로 렌더링되며, 경로가 겹쳐도 z-index 충돌 없이 각자 표시된다.
- **레이아웃**: `ZoneFooter`를 해체하고, `RunnerDiamond`를 독립 컴포넌트로 분리한다. 게임뷰 좌측 패널은 스트라이크존 → 러너다이아몬드 → 볼카운트 순으로 재배치하며, 다이아몬드 크기는 220×200px 이상으로 키운다.

## UX Acceptance Criteria

**UC1. 기본 렌더링**
- 다이아몬드가 스트라이크존 아래, 볼카운트 위에 표시된다
- 주자가 없을 때 베이스는 빈 사각형(outline)으로 표시된다
- 주자가 있는 베이스에는 흰색 도트가 표시된다

**UC2. 이동 애니메이션**
- `runner_advance` 이벤트 발생 시 도트가 from 베이스에서 to 베이스로 루 경로를 따라 이동한다
- 이동 시간은 0.5s이며 플레이백 속도와 무관하다
- 타자 도트는 홈플레이트 아래에서 출발한다

**UC3. 아웃 처리**
- 아웃 이벤트 발생 시 도트가 현재 위치에서 즉시 페이드아웃한다 (0.3s)

**UC4. 도루**
- `steal_attempt` 시 도트가 목표 베이스 방향으로 이동을 시작한다
- `steal_result` 성공 시 도트가 목표 베이스에 도착한다
- `steal_result` 실패 시 도트가 현재 위치에서 즉시 페이드아웃한다

**UC5. 동시 이동**
- 여러 주자가 동시에 이동할 때 각 도트가 독립적으로 움직인다
- 도트 간 겹침 없이 각자 베이스 위치에 표시된다

## Requirements

### Must-have

**R1. RunnerDiamond 컴포넌트**
- 기존 `ZoneFooter`의 소형 다이아몬드를 제거하고 독립 컴포넌트 `RunnerDiamond`로 교체
- 크기: `min(220px, 70vw)` 반응형 적용, 비율 220×200 유지
- 베이스 경로(홈→1루→2루→3루→홈)를 선으로 표시

**R2. 주자 도트 렌더링**
- 각 루를 점유한 주자마다 흰색 원형 도트 렌더링 (Player 데이터 모델 변경 없음, 단일 색상)
- 도트는 해당 베이스 좌표에 정적으로 위치

**R3. runner_advance 애니메이션**
- `runner_advance` 이벤트 발생 시 각 move의 from → to 경로를 CSS transition으로 이동 (0.4~0.6s)
- 타자는 홈플레이트 근처('batter' 위치)에서 출발
- 홈인(to === 'home') 시 홈플레이트 도달 후 페이드아웃

**R4. 도루 애니메이션 + 플레이백 단위 처리**
- `steal_attempt`: 도루 시도 시 도트가 목표 베이스로 이동 시작 (1칸, 경유 없음)
- `steal_result`: 성공 → 목표 베이스 도착, 실패 → 목표 베이스 도달 후 페이드아웃
- **타석 단위 진행 시**: 타석 중간에 도루가 발생한 경우, `steal_attempt` + `steal_result`를 별도 단위로 먼저 공개 후 나머지 타석을 이어서 진행. 즉, `nextUnitEnd` 함수가 `at_bat_result` 이전에 `steal_attempt`를 만나면 `steal_result` 직후를 중간 경계로 처리한다.
- **투구 단위 진행 시**: 이벤트 순서대로 자연스럽게 처리됨 (기존 동작 유지)

**R5. 태그업 애니메이션**
- `tag_up` 이벤트: safe → 목표 베이스로 이동, out → 즉시 페이드아웃

**R6. 아웃 공통 처리**
- 모든 아웃(도루 실패, 태그업 아웃, 송구 아웃) → 아웃 이벤트 발생 즉시 페이드아웃

**R7. 레이아웃 재배치**
- 게임뷰 좌측 패널: 스트라이크존 → RunnerDiamond → 볼카운트+아웃카운트 순 세로 배치
- 볼카운트(B/S)와 아웃 카운트는 RunnerDiamond 하단에 인접 배치

### Nice-to-have

**N1. 홈인 pulse 효과**
- 홈인 도트 페이드아웃 직전 짧은 pulse 효과

**N2. 주자 이름 툴팁**
- 도트 hover 시 선수 이름 표시 (Runner 객체를 liveState에서 노출해야 함)

## Success Definition

- 주자가 루 이동 시 도트가 루 경로(1루 방향, 2루 방향, 3루 방향, 홈 방향)를 따라 시각적으로 이동한다
- 도루 성공/실패, 태그업 모두 애니메이션이 적용된다
- 여러 주자가 동시에 이동(예: 만루에서 안타)할 때 각 주자가 독립적으로 움직인다
- 애니메이션 속도는 플레이백 속도와 무관하게 일정하다 (항상 0.4~0.6s 내에 완료)
- 기존 볼카운트·아웃 카운트 표시는 그대로 유지된다
